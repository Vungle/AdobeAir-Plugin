create()method
public static function create(appIds:Array, logCallback:Function = null, endpoint:String = null):Vungle

Initializes the Vungle Extension. Vungle is a Singleton and you may have only one instance. After creation, you may access the instance through the static property Vungle.vungle .

Parameters

appIds:Array — Array of appIds targeted by this application. If targeting a single platform (iOS or Android), pass the string App ID from the Vungle dashboard alone ( ["ios_id"] or ["android_id"] ). If targeting multiple platforms, pass the iOS id as the first element, and the Android id as the second ( ["ios_id", "android_id"] ).
 
logCallback:Function (default = null)

Returns
Vungle — initialized instance of Vungle extension interface.

displayIncentivizedAd()method 
public function displayIncentivizedAd(name:String = null, showCloseButton:Boolean = false, orientationHint:int = 0):void
Deprecated: Please Use playAd()

If you are interested in including incentivized view mechanics into your game or application, you can use this method to display an incentivized video ad. First, you should check isAdAvailable() first to determine if an ad has been downloaded and is available for play. Determine if the user is economically viable to be monetized via incentivized views. Offer a video view for some virtual currency, i.e. 'Watch this 15 second video and receive 5 gold coins!'. In the VungleEvent.AD_VIEWED listener, if the watched value is 80% of the length property value, then you will be paid for the video view, so your currency can be rewarded to this user- and you can display UI notifying the user they have received in-game currency.

Parameters

name:String (default = null) — set this property if you have a String to identify your user with. This can be used in our server-side webhooks, which allow you to trigger an HTTP GET whenever a user completes a view.
 
showCloseButton:Boolean (default = false) — ignored. Use the forced view options in your app's advanced settings on the Vungle Dashboard.
 
orientationHint:int (default = 0) — orientation of the ad.

isAdAvailable()method 
public function isAdAvailable():Boolean

Checks whether the Vungle extension has an ad available to be played. When your app has reached a suitable stopping point, you can ask the VunglePub SDK whether it has an ad ready to be displayed using this method. If it returns true, than at least one ad has been completely downloaded to disk and is ready to display. If it returns false, there are no video ads to dsiplay at this time. Note that this method will only return true when the device is connected to the internet. You should check isAdAvailable() before calling displayAd() or displayIncentivizedAd().

Returns
Boolean — true if an ad has been downloaded and is ready to play. False otherwise.

AD_VIEWED Event  
Event Object Type: com.vungle.extensions.events.VungleEvent
VungleEvent.type property = com.vungle.extensions.events.VungleEvent.AD_VIEWED

Dispatched when an ad has been viewed. The watched and length properties of the event will be populated with the amount of the video watched, in seconds, and the total length of the video in seconds, respectively, which are useful for rewarding incentivized views. This event may not be called in some cases, such as when there is a pre-roll HTML asset in the advertisement and the user opts out of the ad before seeing the video. Listen for VungleEvent.AD_FINISHED to capture an event whenever an ad has been dismissed.
